# 分块建图

当一张地图因过大而无法保存在单个文件中时，我们可以创建几张相互连接的地图。当机器人在两张地图的重叠区域时，它可以切换地图并继续运行。

经过测试，各个版本支持的单张地图面积为：

| 底板版本 | 建图面积  |
| -------- | ------ |
|  2.14 | 200-300万平米 |
|  2.13 或更早 |  20-30万平米 |

## 分块建图的难点

大场景下分块建图。要保证：

1. 多张地图的坐标系完全一致。
2. 公共部分完全重合。

:::warning
室外建图，建议打开 GPS 或者 RTK。这样就不用什么技巧，基本都能成功回环。
如果是纯室内，则需要一些技巧，才能保证各区域连接正确。
:::

两个方法：

* 线性建图法
* 骨干优先法

## 方法1：线性建图

本方法，适合多个线性相连的区域。

![三个区域，之间只有单通道相连|600](./img/suitable.png)

步骤：

1. 首先，创建区域 1 的地图。
2. 在区域 1 内进行定位，并移动到区域 1 和区域 2 的重叠区域。
3. 开始创建新地图，确保将 `start_pose_type` 设置为 `"current_pose"`。这确保使用当前坐标作为新地图的起点，以保持坐标系之间的连续性。
4. 区域 2 创建完成后，重复此过程。
5. 根据需要继续创建区域 3 及后续区域。

您可以通过在开始新地图之前略微后退到前一个区域来增加地图之间的重叠部分。更大的重叠范围为地图切换提供了更稳健的区域。

### 线性建图的局限性

此方法仅在起点处继承前一张地图的坐标。由于地图之间没有匹配或回环检测（闭环），因此仅能保证在起点附近的对齐。

具有清晰单通道过渡点的地图非常适合此方法。
如果两部分之间有多个通道连接，则不建议使用这种简单的方法。

## 方法2：骨干优先法

骨干（Backbone）方法支持多通道场景，并确保所有通道均实现正确的对齐。

### 骨干分块建图步骤

1. 首先，分析并规划布局，以确定骨干和各个单独区域。骨干应包括主要路线和大型环路，并连接到所有相关区域。
2. 沿着规划好的路线行走创建骨干地图，并可命名为 "backbone"（骨干）。骨干成功建立后，就保证了整体形状的正确性。
3. 加载骨干地图，导航到区域 1 附近，并开始增量建图。
4. 完成建图时，确保将 `new_map_only` 参数设置为 `true`。这确保仅保存新增量的区域，而不重复保存骨干地图数据。
5. 对后续每个区域重复此过程。
