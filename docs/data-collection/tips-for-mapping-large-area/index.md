# 绘制大面积地图的技巧

## 摘要

本文旨在指导您如何在大型场景中，顺利完成高效且准确的建图任务。

我们还深入解析了地图数据大小的影响因素。
掌握这些知识，有助于您养成更合理的规划与走位习惯，
不仅能有效减少不必要的 CPU 与内存消耗，
更能构建出结构稳定、覆盖完整的超大规模地图。

## 地图数据由什么决定？

地图并不等同于简单的 2D 图片，
其底层实则包含了密集的点云、子图（Submaps）和机器人的历史运动轨迹等信息。
因此，单纯的“地理占地面积大”并不意味着最终的地图文件也会很大。

我们可以对比以下三种典型大面积场景的数据量差异：

* **场景 1（数据稀疏）**：大型厂房外围道路。
  大部分厂区内部不可进入，雷达实际上只能扫到厂区周围的一圈道路。
  虽然包含的空间极广，但被扫描到的特征面积占比很低（可能仅有 10%-20%）。
  这类地图的数据量往往**较小**。
* **场景 2（数据中等）**：空旷的工厂内部。
  四周视野极好，激光轻松就能扫描到约 70% 的面积。
  由于没有纵横交错的货架阻挡，机器只要随意走动几圈就能实现全面覆盖。
  这类地图文件的大小通常**适中**。
* **场景 3（数据密集）**：满排货架的密集仓储区。
  虽然长宽不够大，但密布着细长的过道。
  机器人必须耗费极长时间，像走迷宫般穿梭于几乎所有的狭小缝隙间。
  超长的轨迹线加上极高密度的点云，会导致此类地图的数据量**非常庞大**。

由此可见：场地的物理占据面积，只会在建图期间略微影响 UI 界面的“渲染性能”；
**真正决定地图文件体积的，是您的扫描总时常、行走轨迹长度，以及最终记录的子图特征密度。**

## 大场景建图的核心技巧

:::tip 强烈建议配置绝对定位
在进行室外大场景建图时，极力推荐开启 GPS 或 RTK 功能。
有了绝对定位数据的辅助，系统能有效消除长距离造成的漂移累积，
您将完全不需要再为大面积的回环（闭环对齐）问题而担忧。
本文下方介绍的建图走位规则与技巧，主要是针对纯室内，或无 GPS 信号等特殊场景的。
:::

### 1. 优先建立主干骨架，再补充细节

如果是非常辽阔的大场景，务必**必须先一口气走完外围（搭建主干骨架），再回头慢慢补全中间内部的细节**。
这不仅能保障地图外框形状的准确不畸变，更是避免闭环发生错乱最稳当的做法。

原因在于，IMU 传感器在长时间运行中会无可避免地产生漂移累积。
假设场景有一条数百米长的外环道，如果您走一步停一步，花了一个小时才绕行一圈试图与起点合并，
由于时间过长产生的累积误差，基本会导致闭环时地图被大力拉扯变形。

应对此类大环线地形的操作要诀如下：

1. 建图出发前，确认**校准 IMU 偏移值**与陀螺仪缩放比例。
2. 以适中且连贯的速度，“抄外围”尽早完成外环线探索，**在最短时间内达成骨干闭环**。
3. 一旦大骨架成型、全图坐标系被锚定稳固后，再进入内部圈圈绕路，深入补充小过道和障碍细节。

### 2. 严禁无意义的 S 形走位或原地打转

有些用户在扫描开阔地带时，倾向像吸尘器扫地一样的“S”型反复游走。
这并不能让系统得到更多的环境特征，反而会产生极大比例的冗余特征与重叠轨迹，
导致地图文件无端变卡、子图数量急剧增加。

### 3. 规划合理路线，避免高频走回头路

动手之前，最好是在脑中简略打个草稿——规划出您的最优遍历路线。
适当选取连接处做短段距离的“闭合”，即可达到校正精度的目的。
在实际情况下，难免会重复走过某些关键走廊；
但如果您在评估时发现走过的新路线有 **超过 20%-30%** 往往都是已经走过的老路，
那么路线的结构就必须考虑重置规划了。

### 4. 控制单次建图规模，善用增量建图

建议不要试图一口气扫描完毕整个超大场景。
无论是以时间（如单次不超过 1 小时）、还是以面积与行走距离来衡量，
都应当刻意控制单次建图任务的规模。

这样做的核心目的是**降低试错成本（容错率）**。
如果您连续走了好几公里去建一个庞大的地图，一旦在最后时刻发生闭环失败、
操作失误或意外断电，所有的努力都会前功尽弃，几十万平米的路都白走了。

正确的破局之道是运用“分治法”：
您可以先快速把主要干道和骨架图扫完，果断点击保存并完成建图。
之后使用系统的 **增加扩展地图（增量建图）** 功能，
分作几个独立的任务，逐个区域将细节补全。
这样一来，即使某次局部的增量建图不理想或者失败了，
您也只需重新执行那一次小任务即可，而绝不会丢失之前稳定积累的地图成果。
